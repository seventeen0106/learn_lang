# LangGraph

</div>

> 專為構建狀態化、多角色應用程式而設計的庫

作為 LangChain Library 的擴展，LangGraph 完全兼容 LangChain 生態系統，通過創建有向圖來簡化開發過程，將你的整個工作流程視覺化為一個圖形。 

- 每個任務（如檢查訂單、生成郵件、處理退款）都是圖中的一個【節點】
- 節點之間的【連接（邊）】定義了任務的執行順序和條件
- 整個系統共享一個【狀態】,使得信息可以在不同任務間無縫傳遞

使用 LangGraph,你可以輕鬆定義複雜的條件路徑（例如,只有在訂單延遲時才發送道歉信）,創建循環（比如在客戶不滿意時重複提供解決方案）,甚至並行執行多個任務。
    
## ❓ LangChain vs LangGraph

> LangGraph 並不是要取代 LangChain，而是對其進行補充和擴展。

| 比較項目 | LangChain | LangGraph |
|----------|------------|------------|
| 工作流結構 | 主要是線性的，適合順序執行的任務 | 基於【圖】的結構，適合複雜、非線性的工作流 |
| 靈活性 | 在處理條件分支和循環時較為受限 | 可以輕鬆定義條件【路徑、循環和並行】任務 |
| 狀態管理 | 跨多個步驟維護狀態較困難 | 提供【共享狀態系統】，方便在整個工作流中傳遞資訊 |

## 0. 三大關鍵概念：網狀結構、狀態管理以及協調能力

### 圖結構（Graph Structure）

採用有向圖（Directed Graph）來描述應用程式結構

將每一個節點(Node) 都代表一個代理人，邊(edge) 則是代理人之間傳遞資訊的通道

- 節點（Node）：代表各個智能代理
    
- 邊（Edge）：代表代理之間的信息傳遞通道
    

### 狀態管理（State Management）

即時更新自己的狀態，確保系統能夠維持上下文的連貫性，並對輸入做出最恰當的回應。

- 維護上下文
    
- 對輸入做出適當回應
    
- 在多次交互中追踪和保留關鍵信息
    
### 協調（Coordination）

協調功能極大地簡化了複雜多代理系統的開發流程，使得構建高度智能化的應用變得前所未有的簡單。

- 代理按正確順序執行任務
    
- 信息在代理間無縫交換
    
- 開發者可專注於應用程序的高階邏輯

## 1. LangGraph 的核心元件

1. **圖形(Graph)**: 這是 LangGraph 的核心，就像一張路線圖，包含了所有的節點和它們之間的連接。

2. **狀態(State)**: 想像這是一個共享的記事本，記錄了整個過程中的重要信息。每個節點都可以讀寫這個記事本。

3. **節點(Node)**: 這些是圖中的各個站點，每個站點都代表一個特定的任務或檢查點。

4. **邊(Edge)**: 這些是連接節點的路徑，決定了信息如何從一個節點流向另一個節點。

LangGraph 就像一個郵件配送中心。郵件（狀態）在不同的工作站（節點）之間傳遞，根據特定的規則（邊）決定下一步去哪裡。當整個系統啟動時，所有的節點都在等待，直到收到相應的資訊才開始工作。

這種設計讓我們能夠構建出既靈活又強大的 AI 工作流程。

## 2. 應用範例

### 1\. 智能客服系統

- 利用多個 LLM 代理處理多樣化用戶查詢
    
- 無縫切換對話主題，保持上下文連貫
    
- 狀態管理確保一致的用戶體驗
    

### 2\. 工作流程自動化

- 智能代理可處理文檔處理、審批流程、數據分析等任務
    
- 清晰定義的工作流程減少人工干預
    
- 狀態管理功能確保複雜操作序列的順暢執行
    

### 3\. 多代理協作系統

- 適用於供應鏈管理等複雜系統
    
- 協調能力確保代理間有效溝通和同步決策
    
- 提升整體系統性能和運營效率
    

### 4\. 智能推薦系統

- 多代理分析用戶行為、偏好和上下文數據
    
- 整合多種數據源和算法
    
- 提供精準、相關的產品、內容或服務推薦
